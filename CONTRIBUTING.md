# Contributing

Libre Nikki is a collaborative project designed in a way to be continously extendable. Contributors can extend the game by adding their own maps. Main principle of Libre Nikki is to be as developer-friendly as possible, so even people without any kind of programming knowledge could contribute to the project. There are various ways to contribute to Libre Nikki. Code, map additions, music, spritesheets, tilesets, parallax backgrounds and sound effects are all accepted.

Every contribution is welcome.

If you have any questions, Libre Nikki's project leader can be contacted via mail at <librenikki@proton.me>.

## Pull requests / Asset submissions

It is recommended to contribute by creating a [pull request](https://github.com/libre-nikki-devs/libre-nikki/pulls) on the [Libre Nikki's Repository page](https://github.com/libre-nikki-devs/libre-nikki).

Alternatively, contributions can be sent to the [Libre Nikki's Proton Drive](https://drive.proton.me/urls/G8J79CRFWM#IOwhoWDpzTpV).

If you are submitting assets, you should include all necessary information, such as author's name or license information along with your contributions. For Proton Drive contributions, please upload them in a single ZIP file for convenience.

## Bug reports

If you find any bugs or have a suggestion, feel free to [open an issue](https://github.com/libre-nikki-devs/libre-nikki/issues/new). Before opening a bug report, please search the [issues page](https://github.com/libre-nikki-devs/libre-nikki/issues) and make sure it is not a duplicate.

## Licensing

Libre Nikki's source code is licensed under the terms of the GNU General Public License Version 3 and it is required that all content contributed to Libre Nikki is licensed under a compatible license. See <https://www.gnu.org/licenses/license-list.html> for the license compability with the GPL-3.

All code should be licensed under the terms of the GPL-3 license and for the assets, it is recommended to submit them under the terms of the Creative Commons Attribution 4.0 license (CC-BY-4.0) or keep the assets in public domain.

Note, that all content licensed under a license that is not compatible with the GPL-3, cannot be contributed to Libre Nikki. In this case, if you are the copyright holder of the content, you will have to relicense it to the compatible license, in order to contribute it to Libre Nikki.

While you contribute assets, you should specify in the [ASSET_LICENSING](./ASSET_LICENSING) file what license your content is using and add a copyright notice. Format of the [ASSET_LICENSING](./ASSET_LICENSING) file should look as follows:

```
Copyright (C) <year> <author> <mail (optional)>. All rights reserved.

Following files are licensed under the terms of the
<Full License Name>:

<filename1>, <filename2>.
```

Make sure that a single line does not exceed 80 characters.

By default, if no license is provided, it is assumed that contributed content is in public domain. If your content is in public domain, you do not have to include anything in the [ASSET_LICENSING](./ASSET_LICENSING) file.

Direct Libre Nikki's contributors will be added to the [AUTHORS](./AUTHORS) file unless the contributor wishes to remain anonymous.

## Code

All source code should be located in the `scripts/` directory.

Only GDScript is supported. Code written in other languages (such as C#) will not be accepted.

Every source code file (maps' scripts are the only exception) should include the GPL-3 copyright notice at the top of the file.

It is recommended to follow conventions of the [official GDScript style guide](https://docs.godotengine.org/en/stable/tutorials/scripting/gdscript/gdscript_styleguide.html).

## Assets

Assets featuring copyright-infringing, low quality, AI-generated, pornographic, political, religious or execessively violent content will not be accepted.
